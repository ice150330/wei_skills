# Normal Mode

## 目的

用于单一功能、组件或明确 bug 修复。先做简短规划，再集中执行，避免过度流程化。

## 适用场景

- 影响 1-3 个文件。
- 单一功能或组件。
- 需求清晰。
- 已有代码库上下文。
- 预计 10 分钟到 4 小时完成。

## 执行流程

```
用户请求
    ↓
简短规划
    ↓
加载专家
    ↓
执行
    ↓
快速审查
    ↓
完成
```

## 响应模板

```
这是一个中等复杂度任务，我会使用 [expert] 模式：

简短计划：
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

[执行]

完成
```

## 专家选择

| 任务类型 | 主专家 | 辅助专家 |
|----------|--------|----------|
| UI 组件 | Dev | Designer |
| API 接口 | Dev | QA |
| 数据库查询 | Dev | Architect |
| 表单校验 | Dev | QA |
| Bug 修复 | 领域专家 | QA |

## 简短规划模板

```
1. 理解：具体要完成什么？
2. 方法：准备如何实现？
3. 文件：会修改哪些文件？
4. 风险：可能出什么问题？
```

## 规则

1. 只做简短规划，不扩大范围。
2. 最多启用 1-2 个专家。
3. 聚焦执行，减少说明性文字。
4. 只在必要时更新 README 或相关文档。
5. 完成前做快速自检。

## 模式切换

### 升级到 Deep Mode

触发条件：
- 影响文件超过 5 个。
- 出现架构决策。
- 发现复杂集成。
- 预计超过 4 小时。
- 需要超过 2 个专家。

说明模板：

```
升级到 Deep Mode：原因
```

### 降级到 Quick Mode

触发条件：
- 任务比预期简单。
- 只影响 1 个文件。
- 解决方案少于 20 行。
- 10 分钟内完成。

说明模板：

```
按 Quick Mode 完成：原因
```

## 示例

### 适合 Normal Mode

- “给表单加邮箱校验。”
- “创建用户列表 API。”
- “修复这个 React 组件 bug。”
- “优化这个数据库查询。”
- “增加 CSV 导出功能。”

### 不适合 Normal Mode

- “构建电商平台。” → 使用 Deep Mode。
- “怎么打印？” → 使用 Quick Mode。
