# 任务评估器

## 目的

在 3 秒内完成复杂度评估，并用置信度分数把任务路由到合适的执行模式。

## 评估因素

```
Score = (keyword_match × 0.4) + (file_complexity × 0.3) + (historical × 0.3)
```

### 1. 关键词分析（权重 40%）

读取 `heuristics.yaml` 中的加权关键词：
- 复杂关键词（如 `platform`、`architecture`、`平台`、`架构`）：+0.8-0.9
- 中等关键词（如 `feature`、`api`、`功能`、`接口`）：+0.5-0.6
- 简单关键词（如 `fix typo`、`print`、`错别字`、`打印`）：+0.1-0.2

### 2. 文件上下文（权重 30%）

```
if git_status shows:
  - 1 个文件，<50 行 → 简单
  - 2-5 个文件，50-500 行 → 中等
  - 5+ 个文件，>500 行 → 复杂
```

### 3. 历史准确度（权重 30%）

跟踪过往评估结果：
- 同类任务之前是否评估准确
- 根据实际执行时间和文件影响范围调整判断

## 置信度等级

| 等级 | 范围 | 动作 |
|------|------|------|
| 高 | >0.8 | 直接路由，无需确认 |
| 中 | 0.5-0.8 | 简短向用户确认 |
| 低 | <0.5 | 先询问澄清问题 |

## 决策流程

```
用户输入
    ↓
┌────────────┐
│ 计算分数   │
│ 读取规则   │
└─────┬──────┘
      ↓
┌────────────┐
│ 检查置信度 │
└─────┬──────┘
      ↓
 ┌────┴────┐
 高       中/低
 │         │
 ▼         ▼
路由    确认/询问
 │         │
 └────┬────┘
      ↓
  激活模式
```

## 反馈循环

任务完成后：
1. 记录实际复杂度。
2. 对比初始评估。
3. 必要时调整权重。
4. 更新历史准确度。

## 评估标准

### Quick Mode（少于 10 分钟）

**判断信号：**
- 单个命令或单个问题
- 少于 20 行代码
- 已有明确解决方案
- 不需要设计决策

**示例：**
- “Python 怎么打印？”
- “修复这个错别字”
- “运行测试”
- “解释这个错误”

**动作：** 直接回答或执行，不启动完整工作流。

### Normal Mode（10 分钟 - 4 小时）

**判断信号：**
- 影响 1-3 个文件
- 单一功能或组件
- 需求清晰
- 有现有代码上下文

**示例：**
- “给表单加邮箱校验”
- “创建用户列表 API”
- “修复这个 React 组件 bug”
- “优化这个数据库查询”

**动作：** 简短规划，按需启用 1-2 个专家。

### Deep Mode（4 小时以上）

**判断信号：**
- 影响 4+ 个文件或新项目
- 需要架构决策
- 涉及多个组件或集成
- 需求不清晰或持续变化

**示例：**
- “构建电商平台”
- “重构认证系统”
- “设计实时聊天功能”
- “创建新的移动应用”

**动作：** 完整规划，多专家分阶段执行。

## 快速决策树

```
1. 是否是关于现有代码的简单问题？
   是 → Quick

2. 是否是单文件、少于 50 行的修改？
   是 → Quick

3. 是否需要设计或架构决策？
   是 → 跳到 5

4. 是否跨多个模块或需要协调？
   是 → Deep
   否 → Normal

5. 是否是新项目或大型重构？
   是 → Deep
   否 → Normal
```

## 响应模板

### Quick 响应

```
这是一个简单任务，我会直接处理：

[直接解决方案]

完成（X 秒）
```

### Normal 响应

```
这是一个中等复杂度任务，我会使用 [expert] 模式：

简短计划：
1. [步骤 1]
2. [步骤 2]
3. [步骤 3]

[执行]

完成
```

### Deep 响应

```
这是一个复杂项目，我会使用 Deep Mode 进行完整规划：

阶段 1：需求与设计
阶段 2：实现
阶段 3：质量保证
阶段 4：部署

[开始阶段 1]
```

## 上下文感知

评估器会考虑：
- 当前目录是否存在 `CLAUDE.md`
- Git 状态：已有代码库还是新项目
- 相关文件类型
- 用户历史偏好
- 当前对话上下文

## 动态调整

如果执行过程中发现复杂度变化，应及时切换模式：

```
初始评估为 Normal，但发现：
- 需要数据库迁移
- 需要 API 版本管理
- 影响 5+ 个文件

→ 升级到 Deep Mode
```
